Detected by the Font Awesome agent tools. Commit this file so the whole team benefits.

## Integration

- **Method:** Build-time npm imports via `@fortawesome/fontawesome-svg-core` (tree-shaken by Vite), no Kit or CDN
- **Version:** 7.3.0
- **License:** Pro
- **Families:** classic (solid), duotone

## Defaults

- **Style:** solid
- **Family:** classic

## Import Pattern

All icons live in a single registry module: `frontend/modules/ui/icons.js`. To add an icon:

1. Import it from `@fortawesome/pro-solid-svg-icons` (or `@fortawesome/pro-duotone-svg-icons` for duotone)
2. Add it to the `library.add(...)` call
3. Add a kebab-case entry to `iconDefinitions` (or `duotoneDefinitions`)

Icons are then used two ways:

- **Web Awesome components:** `<wa-icon name="cloud-sun">` (default library) or `<wa-icon library="duotone" name="poo-storm">` — resolved via `registerIconLibrary` in `icons.js`
- **Canvas rendering:** `drawWeatherIcon(ctx, 'fa-cloud-sun', x, y, size)` in `frontend/modules/cards/core.js`, which strips the `fa-` prefix and looks up path data via `getIconData(name)`

## Conventions

- Private npm registry auth for `@fortawesome` scope via `.npmrc` + `FONTAWESOME_NPM_TOKEN` in `.dev.vars`; export before `npm install` (see AGENTS.md)
- Keep imports, `library.add`, and `iconDefinitions` alphabetized and in sync (icon count noted in the comment atop `icons.js`)
- Duotone icons render with secondary path at 40% opacity; mutator sets `fill="currentColor"`
